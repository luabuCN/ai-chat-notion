# 文档权限与协同机制设计

## 1. 权限模型总览

系统包含两种文档权限来源：

### 空间权限（继承权限）

文档归属于一个空间。用户加入空间后，通过空间角色获得默认权限：

| 角色 | 默认权限 | 可做的事 |
| --- | --- | --- |
| 管理员 | 可编辑 | 管理空间、成员、权限；创建、编辑、删除文档。 |
| 编辑成员 | 可编辑 | 创建、编辑、删除文档；评论；分享（受空间设置限制）。 |
| 查看成员 | 可查看 | 查看文档内容；评论（如果开放评论）。 |

### 单文档分享权限（直接授权）

对单个文档可单独分享给外部用户或空间成员：

| 权限 | 说明 | 可做的事 |
| --- | --- | --- |
| 可编辑 | 可编辑文档内容 | 编辑内容、评论、参与协同（满足协同条件时）。 |
| 可查看 | 仅查看文档内容 | 查看内容、评论（如果开放评论）；不能编辑，不能参与协同。 |

分享只影响单个文档，不改变文档归属，也不改变用户的空间角色。

## 2. 最终生效权限

当用户同时拥有空间权限和单文档分享权限时，最终权限取更严格的那个：

```ts
finalPermission = min(spacePermission, documentSharePermission)
```

权限等级：

```ts
无权限 < 可查看 < 可编辑
```

规则示例：

| 空间权限 | 单文档分享权限 | 最终生效权限 | 说明 |
| --- | --- | --- | --- |
| 可编辑 | 可编辑 | 可编辑 | 一致时保持不变。 |
| 可编辑 | 可查看 | 可查看 | 取更严格权限。 |
| 可查看 | 可编辑 | 可查看 | 取更严格权限。 |
| 可查看 | 可查看 | 可查看 | 一致时保持不变。 |
| 无权限 | 可编辑 | 可编辑 | 非空间成员通过分享获得权限。 |
| 无权限 | 可查看 | 可查看 | 非空间成员通过分享获得权限。 |

文档所有者和空间所有者拥有 owner 权限，属于管理特权，不参与 `min` 限制。

## 3. 协同开启条件

同时满足以下条件时，开启多人实时协同：

1. 文档开启实时编辑能力。
2. 至少 2 个用户同时处于编辑状态。
3. 参与者的最终生效权限为可编辑或 owner。
4. 所有编辑者与协同服务保持正常连接。

协同状态规则：

| 条件 | 状态 |
| --- | --- |
| 编辑人数 >= 2 | 显示“协同中 + 人数”，实时光标和内容同步。 |
| 编辑人数 = 1 | 关闭协同标签，单人编辑。 |
| 所有人离开编辑态或只剩 1 人编辑 | 协同关闭。 |
| 只读用户进入 | 不进入协同，只能查看或评论。 |

## 4. 权限与协同关系

| 用户身份 | 拥有权限 | 是否可参与协同 |
| --- | --- | --- |
| 空间成员 | 可编辑 | 满足协同条件时可以参与协同。 |
| 外部用户 | 可查看 | 不能参与协同，仅可查看或评论。 |

关键点：只有最终生效权限为可编辑或 owner 的用户，才允许进入协同编辑。

## 5. 典型场景

| 场景 | 空间权限 | 分享权限 | 最终权限 | 协同结果 |
| --- | --- | --- | --- | --- |
| 空间成员可编辑，未被单独分享 | 可编辑 | 无 | 可编辑 | 可编辑，可参与协同。 |
| 空间成员可编辑，被单独分享为可查看 | 可编辑 | 可查看 | 可查看 | 只能查看，不能参与协同。 |
| 空间成员可查看，被单独分享为可编辑 | 可查看 | 可编辑 | 可查看 | 只能查看，不能参与协同。 |
| 非空间成员，被分享为可编辑 | 无 | 可编辑 | 可编辑 | 可编辑，满足条件可协同。 |
| 非空间成员，被分享为可查看 | 无 | 可查看 | 可查看 | 只能查看，不能参与协同。 |

## 6. 展示规则

文档列表或详情页只展示最终生效权限和协同状态：

- 权限：可编辑 / 可查看。
- 状态：协同中（仅在编辑人数 >= 2 时显示）。

示例：

```text
Vite 权限设计方案
[可编辑] [协同中 2人]

产品路线图
[可查看]
```

## 7. 结论

- 文档只归属于一个空间。
- 空间权限是继承权限，单文档分享权限是直接授权。
- 同时存在空间权限和单文档分享权限时，最终权限取更严格值。
- 协同是运行状态，不是权限来源。
- 只有最终生效权限为可编辑或 owner 的用户，才能参与协同编辑。
